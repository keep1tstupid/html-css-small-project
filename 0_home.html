Here you hopefully can find more or less useful information about programming.

In the Haaga-Helia University of Applied Sciences according to Orientation to Digital Services course programme, every student should present his/her individual web-project about programming. I hope you will have a nice time checking my work. We had two files with guidelines to follow, but also short version is available in our Moodle system. Below you can find these guidelines and my short explanations: how my project meets them.

There are no strict rules, what is very important and what is not. Some simple rule of thumb:

  1. MOST IMPORTANT: HTML and CSS validity, your code is readable and logical
  2. Different layouts
  3. Semantics: your code is machine-readable.
  4. Usability: your site is readable and logical to navigate.
  5. Content: your text and pictures are logical to understand.
  6. LEAST IMPORTANT: Colors, Fonts, Fancy graphics, Animations, JavaScript.

1. HTML and CSS validity

You can use links from the footer to make sure that everything is fine! Validators are available on every page. Unfortunately links work only with university environment.

2. Different layouts

Firstly in my project, I used just normal CSS for "Home page" & "Materials", grid layout for "Eureka" & "Next steps" pages and flexbox layout for "Jobs" and "Contact" pages. After the first teachers review all pages were linked to flex-layout (to make it look more consistent). Grid container included on "Jobs" page.

3. Semantics

I tried to include semantic elements as much as possible. In my HTML files you can find following elements: <header>, <nav>, <section>, <aside>, <footer>.

4. Usability

I tried to make navigation obvious (selected menu item is highlighted, page title in the header), content understandable and well-structured. Firstly I made my colour-scheme minimal and design simple. After completing layout fixes, I changed the design and brought more fun. :)

5. Content

All content structure was taken from requirements. Page "Contact me" is optional, there I decided to create a form and different buttons. For GitHub a lot of "essay" parts removed.

6. Decorations

In the first versions, there was no decoration, grey colours (see screenshot below). After that I have added colours, moving objects and striped background for header and footer. All there changes implemented with SCSS and CSS strips generator.
